Kurulum Süreci

Go 1.23 Kurulumu ve Proje Build Tamamlandı
Go 1.23 kuruldu. Repo klonlandı, bağımlılıklar indirildi. go build -o bin/ch-api ./cmd/ch-api başarıyla tamamlandı.
Sistem Kullanıcısı ve Dizin Yapısı Tamamlandı
ch-api sistem kullanıcısı oluşturuldu. /etc/ch-api/, /var/lib/ch-api/, /var/run/ch-api/ dizinleri yapılandırıldı.
Config ve JWT Secret Yapılandırması Tamamlandı
/etc/ch-api/config.yaml oluşturuldu. JWT secret openssl rand -hex 32 ile üretildi. Auth, rate limiting, cloud_hypervisor, network, images bölümleri yapılandırıldı.
systemd Unit Dosyası Yapılandırması Tamamlandı
/etc/systemd/system/ch-api.service yapılandırıldı. AmbientCapabilities (CAP_NET_ADMIN, CAP_NET_BIND_SERVICE, CAP_NET_RAW, CAP_SYS_ADMIN) eklendi. PrivateTmp=no, ProtectSystem=no ayarlandı.
ext4 Loop Device (Image Volume) Tamamlandı
Btrfs üzerinde Cloud Hypervisor'ın disk imajlarını açamaması nedeniyle /var/lib/ch-api/images.img olarak ext4 loop device oluşturuldu. Ubuntu cloud image, bzImage kernel, CLOUDHV.fd firmware buraya kopyalandı. /etc/fstab'a reboot kalıcılığı için eklendi.
03

Çözülen Kritik Problemler

P-01
VMM Client Bağlantısı Eksikti
main.go'da svc.SetVMMClient() hiç çağrılmıyordu. Boot istekleri "vmm client not configured" hatasıyla başarısız oluyordu.
→ vmm.New() + svc.SetVMMClient() main.go'ya eklendi
P-02
Kernel → Payload Mapping Hatası
CH v51'de kernel doğrudan değil, payload.kernel üzerinden iletilmesi gerekiyordu. Handler KernelConfig'i CH'ye gönderiyordu ama CH bunu reddediyordu.
→ handler'da kernel alanı payload.kernel olarak map edildi
P-03
Btrfs Disk İzin Hatası
CachyOS Btrfs dosya sistemi üzerinde Cloud Hypervisor disk imajlarını yazılabilir açamıyordu. "Read-only file system (os error 30)" hatası alınıyordu.
→ ext4 loop device oluşturuldu (/var/lib/ch-api/images.img)
P-04
TAP Interface İsimlendirme Çakışması
Tüm VM ID'leri aynı prefix'le başladığından TAP isimleri hep "tap-vm177895" oluyordu ve ikinci VM boot edilemiyordu.
→ nextIndex ile tap-1, tap-2, ... formatına geçildi
P-05
Global CH Socket Preflight Hatası
Her VM artık kendi CH process'ini başlatıyor. Ama main.go'daki global socket kontrolü hâlâ /var/run/ch-api/ch.sock'u arıyordu ve servis başlamıyordu.
→ main.go'daki preflight socket kontrolü kaldırıldı
P-06
UFW DHCP Trafiğini Engelliyordu
UFW'nin ufw-after-input zincirinde port 67 (DHCP) ufw-skip-to-policy-input'a yönlendiriliyordu. DHCP sunucusu paketleri göremiyordu.
→ ufw allow in on tap+ port 67/68 kuralları eklendi
P-07
DHCP Server Port 67 Bind Hatası
Go DHCP server port 67'ye bind olamıyordu çünkü servis ch-api kullanıcısıyla çalışıyordu ve CAP_NET_BIND_SERVICE yoktu.
→ systemd unit'e AmbientCapabilities=CAP_NET_BIND_SERVICE eklendi
P-08
ISO9660 Block Size Hatası
go-diskfs kütüphanesi ISO oluşturulurken "blocksize must be one of 2048, 4096, 8192" hatası veriyordu. Doğru SectorSize parametresi gerekiyordu.
→ diskfs.SectorSize512 + LogicalBlocksize=2048 ile düzeltildi
P-09
cloud-init DataSourceNone
cloud-init cidata ISO'yu bulamıyordu ve "Used fallback datasource" mesajıyla devam ediyordu. SSH authorized_keys oluşturulmuyordu.
→ kernel cmdline'a ds=nocloud eklendi, ISO label cidata olarak doğrulandı
P-10
MAC Adresi Uyuşmazlığı
cloud-init ISO'daki network-config'te tanımlanan MAC adresi ile CH'nin VM'e atadığı MAC adresi farklıydı. VM ağ konfigürasyonunu uygulayamıyordu.
→ TAPConfig'e MAC alanı eklendi, CH config'e ve ISO'ya aynı MAC iletildi
P-11
DHCP Offer Kabul Edilmiyordu
VM DHCP Discover gönderiyordu, Offer alıyordu ama Request göndermeden tekrar Discover yapıyordu. Server Identifier (option 54) eksikti.
→ DHCP Offer/Ack'e OptServerIdentifier eklendi; statik IP'ye geçildi
P-12
SSH PasswordAuthentication Override
Ubuntu cloud image /etc/ssh/sshd_config.d/60-cloudimg-settings.conf dosyasında PasswordAuthentication no ile ana config'i override ediyordu.
→ Base image'da 60-cloudimg-settings.conf düzeltildi

05

VM Boot Orchestration Akışı

internal/service/vm_lifecycle.go — BootVM() 
// POST /api/v1/vms/:id/boot çağrıldığında çalışan 11 adımlı süreç

Step 1:  VM config'i store'dan al
Step 2:  Base image'dan VM diski kopyala  → /var/lib/ch-api/images/<vm-id>.raw
Step 3:  TAP interface allocate et         → tap-N, 10.100.N.1/30
Step 4:  SetupTAP                           → ip tuntap add, ip addr add, ip link set up
Step 4b: cloud-init ISO üret               → MAC + statik IP + SSH key
Step 5:  UFW DHCP kuralı ekle              → ufw allow in on tap-N port 67/68
Step 6:  DHCP server başlat                 → goroutine, SO_BINDTODEVICE=tap-N
Step 7:  cloud-hypervisor process başlat   → --api-socket /var/run/ch-api/<id>.sock
Step 8:  Socket bekle                        → max 20 x 300ms poll
Step 9:  VMM client oluştur                  → TransportUnixSock
Step 10: CH'ye VM config gönder             → vm.create (disk + net + serial)
Step 11: Boot et                             → vm.boot

→ "vm fully provisioned" log mesajı

Hızlı VM Oluşturma ve SSH Bağlantısı 
# 1. Token üret
export TOKEN=$(python3 /tmp/gentoken.py)

# 2. VM oluştur
VM=$(curl -s -X POST -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"my-vm","cpus":{"boot_vcpus":2,"max_vcpus":4},"memory":{"size":1073741824}}' \
  http://localhost:8080/api/v1/vms | grep -o '"id":"[^"]*"' | cut -d'"' -f4)

# 3. Boot et (disk + SSH key + cloud-init + TAP + DHCP + CH otomatik)
curl -s -X POST -H "Authorization: Bearer $TOKEN" \
  http://localhost:8080/api/v1/vms/$VM/boot

# 4. SSH key indir
curl -s -H "Authorization: Bearer $TOKEN" \
  http://localhost:8080/api/v1/vms/$VM/ssh-key > vm-key.pem
chmod 600 vm-key.pem

# 5. Bağlan (2 dakika sonra)
ssh -i vm-key.pem ubuntu@10.100.1.2
